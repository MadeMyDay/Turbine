Introduction

CSSP (CSS Preprocessor) is a PHP-powered tool that... CSSP takes something like this … 

BEEINDRUCKENDES BEISPIEL
… and turns it into this: 
BEEINDRUCKENDES BEISPIEL

There is no guarantee, explicit or implied, of its suitability for any purpose. That said, CSSP can save you a lot of time and allow you to concentrate on a website's design and functionality instead of css's limitations and your least favourite browser's latest bugs. And, being fully extensible, you can customize it to your liking.

Goals and Features

CSSP's main goal is a decrease in css devolopment time and web developer headache. This is achieved by a set of basic features that expand what you can do with css:

  • Packing, gzipping and automatic minification of multiple css files
  • Constants (also known as "css variables")
  • Selector aliases
  • Oop-like inheritance, extension and templating features
  • Nested css selectors
  • Fully exensible through a very simple plugin system
  • Built-in device-, browser- and os sniffing

While CSSP's basic features may save you some keystrokes, it's plugins automate and simplify some of the more tedious and complicated aspects of writing css. See the plugins section for details.

Usage

Configuration

The configuration options of CSSP can be set in the stylesheets themselves &ndash a @cssp block (called configuration block) allows for options to be set like properties and values in normal CSS code. There are at the moment two things that can be controlled there: compression and plugins.

Compression

CSSP can output CSS code in a compressed form. 

@cssp {
    compress:1;
}
#foo{
    color: red;
    background: blue;
}

Becomes 

#foo{color:red;background:blue;}

That's a good thing to have since everybody likes smaller downloads and the CSS code generates tends to be more verbose than handwritten CSS. To enable compression simply add compress:1; to your configuration block.

Plugins

@cssp {
    plugins:data_uri border_radius;
}

Aliases

Aliases are constants applied to selectors insted of values. Don't want to remember the complicated #wrapper #header > div ul? You don't have to: 

@aliases {
    mainNavigation: #wrapper #header > div ul;
}

$mainNavigation {
    list-style:none;
}

The @aliases block defines mainNavigation as an alias of #wrapper #header > div ul, which results us to the following css code: 

#wrapper #header > div ul {
    list-style: none;
}

Nested selectors

CSSP implements nested css selectors that work using the new children property: 

#foo div > p {
	color:red;
	font-weight:bold;
	children:{
		a:link, a:visited {
			text-decoration:underline;
		}
		a:hover, a:active {
			text-decoration:none;
		}
		a:focus{
			outline:1px dotted #CCC;
		}
	};
}

Using nested selectors has two possible benefits (provided you don't go overboard using them): they automatically structure your CSSP code into logical units and they can save you a lot of typing. The nested code above compiles to the following css: 

#foo div > p {
	color: red;
	font-weight: bold;
}
#foo div > p a:link, #foo div > p a:visited {
	text-decoration: underline;
}
#foo div > p a:hover, #foo div > p a:active {
	text-decoration: none;
}
#foo div > p a:focus {
	outline: 1px dotted #CCC;
}

Without nested selectors you would have to have typed a lot more #foo div > p. Nested selectors can become a bit confusing if you overuse them, but with a bit of moderation they can save quite a few keystrokes.

Plugins

Border radius [ Lazy ]

Automatically adds vendor-specific versions of border-radius and implements some shortcuts.

Webkit and Mozilla browsers require vendor-specific prefixes for the CSS3 property border-radius. This plugin automatically inserts them wherever a -cssp-border-radius property is found. Additionally, it provides shortcuts for adding rounded corners only on the left, right, top or bottom side of an element: border-left-radius, border-right-radius, border-top-radius, border-bottom-radius

Usage

Add border_radius to your @cssp plugins rule and start using -cssp-border-radius like the standard border-radius CSS3 property.

Examples

Automatic vendor-specific versions

#foo {
    -cssp-border-radius:4px;
}

Result: 

#foo {
    -moz-border-radius: 4px;
    -khtml-border-radius: 4px;
    -webkit-border-radius: 4px;
    border-radius: 4px;
}

Left side only shortcut

#foo {
    -cssp-border-left-radius:4px;
}

Result: 

#foo {
    -moz-border-radius-topleft: 4px;
    -moz-border-radius-bottomleft: 4px;
    -khtml-border-top-left-radius: 4px;
    -khtml-border-bottom-left-radius: 4px;
    -webkit-border-top-left-radius: 4px;
    -webkit-border-bottom-left-radius: 4px;
    border-top-left-radius: 4px;
    border-bottom-left-radius: 4px;
}

Bullet proof @font-face syntax [ Optimization, Lazy ]

Generates bullet proof @font-face declarations from a simplified syntax.

Bulletproof @font-face syntax is a syntax for embedding web fonts in a manner that works for all browsers and takes care of numerous IE quirks. Its only downside is that it's rather complicated: 

@font-face {
    font-family: 'Graublau Web';
    src: url('GraublauWeb.eot');
    src: local('Graublau Web Regular'), local('Graublau Web'),
        url('GraublauWeb.otf') format('opentype')
}

The bullet proof @font-face syntax plugin generates bullet proof @font-face declarations from a simplified syntax.

Usage

  1. Add bp_font_face to your @cssp plugins rule
  2. Build a normal @font-face-rule
  3. Omit the src-attribute
  4. Add properties eot, local and ttf or otf

Example

@font-face {
    font-family:'Graublau Web';
    local:'Graublau Web Regular';
    otf:url('GraublauWeb.otf');
    eot:url('GraublauWeb.eot');
    font-weight:bold;
    font-style:italic;
}

Result: 

@font-face {
    font-family: 'Graublau Web';
    font-weight: bold;
    font-style: italic;
    src: url('GraublauWeb.eot');
    src: local('Graublau Web Regular'), local('Graublau Web'),
        url('GraublauWeb.otf') format(opentype);
}

Hexcrusher [ Performance, Optimization ]

Optimizes hex color definitions.

Hexcrusher finds hex color definition and shortens them whenever possible. A color definition like #FFEE44 would be shortend to #FE4, saving a (tiny) bit of space and loading time.

Usage

Just add hexcrusher to your @cssp plugins rule. Done!

Plugin development

Plugin development for CSSP is rather simple. All you need is a file called plugin_name.php containing a function plugin_name(), located in the plugins directory. The function must accept &$parsed as it's only argument. &$parsed is a reference to the multi-dimensional array containing the parsed CSSP code and it can be freely manipulated by plugins.

Array structure

Array
(
    [@media foo] => Array
    (
        [#bar] => Array
            (
                [property1] => valueA
                [property2] => valueB
            )
    )
    [@media bar] => Array
    (
        [#bar] => Array
            (
                [property1] => valueA
                [property2] => valueB
            )
    )
)

CSS rules that a not part of any @media block a stored in the global block (key css).

Plugin example

In case you wanted your plugin to remove all occurences of background-image from your stylesheets, this would be the code: 

<?php
    /**
     * remove_background_image.php
     * Remove all occurences of background-image (a nonsensical example)
     */
    function remove_background_image(&$parsed){
        // Loop through the blocks
        foreach($parsed as $block => $css){
            // Loop through the selectors
            foreach($parsed[$block] as $selector => $styles){
                // Look for background-image
                if($parsed[$block][$selector]['background-image']){
                    // Be gone!
                    unset($parsed[$block][$selector]['background-image']);
                }
            }
        }
        // No need to return anything
    }
?>

Don't be afraid to do anything CPU- or memory consuming in your plugins - in the end, everything will be calculated only once and be cached afterwards.

To avoid confusion with regular CSS properties, use of the -cssp- prefix for new properties is recommended.

FAQ

Any general advice?

  • Don't use constants only for colors! If you use them for font stacks and paths, you can save yourself a lot of typing – font-family:$serif is so much shorter than font-family:Georgia, "Times New Roman", serif.
  • Don't overuse nested selectors, as they get can get quite confusing.

What are the tradeoffs?

  • CSSP uses the input css files to generate completely new CSS code that won't necessarily look very much like it's source. That means that css debugging tools like Firebug may not be very useful anymore, depending on how many of CSSP's features and plugins you actually use.
  • The CSS selectors generated by CSSP are usually not as efficent as human-written CSS would be.
