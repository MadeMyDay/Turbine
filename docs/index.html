Plugins

Border radius [ Lazy ]

Automatically adds vendor-specific versions of border-radius and implements some shortcuts.

Webkit and Mozilla browsers require vendor-specific prefixes for the CSS3 property border-radius. This plugin automatically inserts them wherever a standard border-radius property is found. Additionally, it provides shortcuts for adding rounded corners only on the left, right, top or bottom side of an element: border-left-radius, border-right-radius, border-top-radius, border-bottom-radius

Usage

Add border_radius to your @cssp plugins rule and start using border-radius (without any prefixes).

Examples

Automatic vendor-specific versions

#foo {
    border-radius:4px;
}

Result: 

#foo {
    border-radius: 4px;
    -moz-border-radius: 4px;
    -webkit-border-radius: 4px;
}

Left side only shortcut

#foo {
    border-left-radius:4px;
}

Result: 

#foo {
    -moz-border-radius-topleft: 4px;
    -moz-border-radius-bottomleft: 4px;
    -webkit-border-top-left-radius: 4px;
    -webkit-border-bottom-left-radius: 4px;
    border-top-left-radius: 4px;
    border-bottom-left-radius: 4px;
}

Bullet proof @font-face syntax plugin [ Optimization, Lazy ]

Generates bullet proof @font-face declarations from a simplified syntax.

Bulletproof @font-face syntax is a syntax for embedding web fonts in a manner that works for all browsers and takes care of numerous IE quirks. Its only downside is that it's rather complicated: 

@font-face {
    font-family: 'Graublau Web';
    src: url('GraublauWeb.eot');
    src: local('Graublau Web Regular'), local('Graublau Web'), url('GraublauWeb.otf') format('opentype')
}

The bullet proof @font-face syntax plugin generates bullet proof @font-face declarations from a simplified syntax.

Usage

  1. Build a normal @font-face-rule
  2. Omit the src-attribute
  3. Add properties eot, local and ttf or otf

Example

@font-face {
    font-family:'Graublau Web';
    local:'Graublau Web Regular';
    otf:url('GraublauWeb.otf');
    eot:url('GraublauWeb.eot');
    font-weight:bold;
    font-style:italic;
}

Result: 

@font-face {
    font-family: 'Graublau Web';
    font-weight: bold;
    font-style: italic;
    src: url('GraublauWeb.eot');
    src: local('Graublau Web Regular'), local('Graublau Web'), url('GraublauWeb.otf') format(opentype);
}

Hexcrusher [ Performance, Optimization ]

Optimizes hex color definitions.

Hexcrusher finds hex color definition and shortens them whenever possible. A color definition like #FFEE44 would be shortend to #FE4, saving a (tiny) bit of space and loading time.

Usage

Just add hexcrusher to your @cssp plugins rule. Done!

Quote style [ Typography, Lazy ]

Inserts language- and country-specific quotation marks.

Different languages and countries use different quotation marks. The quote style plugin introduces a new CSS property called quote-style which allows you to set quotation marks by language and/or country. No more need to remember all the glyphs.

Usage

Include quote_style in your @cssp plugins rule and start using the new quote-style property with the values listed below.

Possible quote-style values

Value Result Glyphs
german #foo { quotes:'\201E \201C \201A \2018'; } „ “ ‚ ‘
german-alt #foo { quotes:'\00BB \00AB \203A \2039'; } » « › ‹
swiss #foo { quotes:'\00AB \00BB \2039 \203A'; } « » ‹ ›
english-uk #foo { quotes:'\2018 \2019 \201C \201D'; } ‘ ’ “ ”
english-us #foo { quotes:'\201C \201D \2018 \2019'; } “ ” ‘ ’

Plugin development

Plugin development for CSSP is rather simple. All you need is a file called plugin_name.php containing a function plugin_name(), located in the plugins directory. The function must accept &$parsed as it's only argument. &$parsed is a reference to the multi-dimensional array containing the parsed CSSP code and it can be freely manipulated by plugins.

Array structure

Array
(
    [@media foo] => Array
    (
        [#bar] => Array
            (
                [property1] => valueA
                [property2] => valueB
            )
    )
    [@media bar] => Array
    (
        [#bar] => Array
            (
                [property1] => valueA
                [property2] => valueB
            )
    )
)

CSS rules that a not part of any @media block a stored in the global block (key css).

Plugin example

In case you wanted your plugin to remove all occurences of background-image from your stylesheets, this would be the code: 

<?php
    /**
     * remove_background_image.php
     * Remove all occurences of background-image (a nonsensical example)
     */
    function remove_background_image(&$parsed){
        foreach($parsed as $block => $css){ // Loop through the blocks
            foreach($parsed[$block] as $selector => $styles){ // Loop through the selectors
                if($parsed[$block][$selector]['background-image']){ // Look for background-image
                    unset($parsed[$block][$selector]['background-image']); // Be gone!
                }
            }
        }
        // No need to return anything
    }
?>

Don't be afraid to do anything CPU- or memory consuming in your plugins - in the end, everything will be calculated only once and be cached afterwards.
